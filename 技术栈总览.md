# 技术栈总览

## 项目定位

本地桌面客户端，用于可视化 Claude Code 的历史对话记录和各项目目录下的交互 Prompt。

## 核心技术栈

| 层级 | 技术 | 版本要求 | 职责 |
|---|---|---|---|
| 桌面框架 | **Tauri v2** | ≥2.0 | 应用壳、窗口管理、系统级文件访问、跨平台打包 |
| 构建工具 | **Vite** | ≥5.0 | 前端构建、HMR 开发体验 |
| 前端框架 | **React** | ≥18 | UI 渲染 |
| 类型系统 | **TypeScript** | ≥5.0 | 类型安全、AI 生成代码质量保障 |
| CSS 方案 | **Tailwind CSS** | ≥3.4 | 原子化样式、快速迭代 |
| 组件库 | **shadcn/ui** | latest | 基于 Radix + Tailwind 的可定制组件 |
| 后端语言 | **Rust** | ≥1.70 | Tauri Commands、文件 I/O、数据解析 |
| 包管理 | **pnpm** | ≥8.0 | 前端依赖管理 |
| Rust 包管理 | **Cargo** | 随 Rust 安装 | Rust 依赖管理 |

## 关键 Rust 依赖 (Crates)

| Crate | 用途 |
|---|---|
| `tauri` | 核心框架 |

## 架构模式

```
┌─────────────────────────────────────────────┐
│              Tauri 窗口 (WebView)             │
│  ┌─────────────────────────────────────────┐ │
│  │         React 前端 (Vite 构建)           │ │
│  │                                         │ │
│  │  Pages ← Components ← Hooks            │ │
│  │                │                        │ │
│  │          invoke('command')              │ │
│  └──────────────┬──────────────────────────┘ │
│                 │ IPC Bridge                  │
│  ┌──────────────┴──────────────────────────┐ │
│  │         Rust 后端 (Tauri Core)           │ │
│  │                                         │ │
│  │  Commands → Services → FileReader       │ │
│  │                              │          │ │
│  │                     ~/.claude/ (数据源)   │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

## 数据流

```
~/.claude/history.jsonl        ──┐
~/.claude/projects/**/*.jsonl  ──┼──→ Rust 解析 ──→ IPC ──→ React 渲染
~/.claude/sessions/*.json      ──┘
```

## 跨平台支持

| 平台 | 打包格式 | WebView 引擎 |
|---|---|---|
| macOS | `.dmg` / `.app` | WebKit (WKWebView) |
| Windows | `.msi` / `.exe` (NSIS) | WebView2 (Chromium-based) |
| Linux | `.AppImage` / `.deb` | WebKitGTK |

## 开发环境要求

| 工具 | 安装方式 |
|---|---|
| Node.js ≥18 | `brew install node` 或 nvm |
| pnpm ≥8 | `npm install -g pnpm` |
| Rust toolchain | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh` |
| Tauri CLI | `cargo install tauri-cli` |
| Xcode Command Line Tools (macOS) | `xcode-select --install` |

## 为什么选这套栈

| 决策点 | 选择 | 原因 |
|---|---|---|
| 为什么不用 Electron | Tauri | 体积小 30 倍（5MB vs 150MB），启动快，适合本地工具 |
| 为什么不用 Next.js | Vite + React | 桌面端不需要 SSR，Vite 更轻更快 |
| 为什么不用 Vue/Svelte | React | AI 训练数据中 React 样本最多，生成质量最高 |
| 为什么后端用 Rust 不用 Node | Tauri 原生 | 读大量 JSONL 文件 Rust 比 Node 快一个数量级，且无需额外运行时 |
| 为什么用 shadcn/ui | 非黑盒 | 组件源码在项目内，AI 可直接修改，不受 npm 包版本限制 |
