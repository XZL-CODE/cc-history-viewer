# 可视化本地 Claude Code 历史对话 — 可行性分析与技术栈选型

## 数据结构一览

| 数据源 | 路径 | 内容 | 规模 |
|---|---|---|---|
| **Prompt 历史** | `~/.claude/history.jsonl` | 每条包含 `display`(用户输入的prompt)、`timestamp`、`project`(工作目录) | 3756 条 |
| **完整对话** | `~/.claude/projects/<编码路径>/*.jsonl` | 每个会话一个文件，含用户/AI消息、工具调用、git分支等完整上下文 | 1548 个会话，407MB |
| **会话元数据** | `~/.claude/sessions/*.json` | pid、cwd、启动时间、CC版本、状态 | 少量活跃session |

## 可行性判断：完全可行

数据结构清晰、本地可读、JSON 格式天然适合前端渲染。能做的可视化维度：

- 按**文件夹/项目**聚合所有历史 prompt
- **时间线**视图（什么时候在哪个项目上最活跃）
- **对话详情**展开查看完整交互
- prompt **关键词搜索**、频率统计
- 按 **git 分支**、**CC 版本**等维度筛选

## 技术栈建议

### 核心原则

选 AI 训练数据中样本最多、社区生态最成熟的栈。AI 写代码本质上是模式匹配，见得多写得好。

### 推荐排序

| 方案 | 组合 | AI Coding 友好度 | 适合场景 |
|---|---|---|---|
| **首选** | **Next.js + TypeScript + Tailwind + shadcn/ui** | 极高（训练样本最密集的现代全栈组合） | 前后端一体，SSR 可直接读本地文件 |
| 备选 | Vite + React + TS + Express 后端 | 高 | 如果不想用 Next.js 的约定式路由 |
| 轻量级 | Python FastAPI + 纯前端 | 中高 | 如果更熟悉 Python |

### 为什么 Next.js 是 AI Coding 的最佳拍档？

1. **TypeScript** — 类型系统让 AI 生成的代码更准确，IDE 自动补全也更好
2. **App Router + Server Components** — 可以在服务端直接 `fs.readFile` 读 `~/.claude/` 下的数据，不需要单独写 API
3. **shadcn/ui** — 基于 Radix + Tailwind 的组件库，AI 对其生成质量极高，且不是黑盒 npm 包，组件代码直接在项目里可改
4. **Tailwind** — 原子化 CSS，AI 一次就能写对样式，不需要来回调

## 项目路径编码规则

`~/.claude/projects/` 下的文件夹命名规则：将工作目录路径中的 `/` 替换为 `-`。

例如：
- `/Users/you/Desktop/我的项目/claude-code` → `-Users-you-Desktop------claude-code`

解析时需要反向还原路径。
