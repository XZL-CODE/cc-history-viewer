# 开发文档

## 0. 产品核心需求

### 核心功能（必须实现）

1. **文件夹 Prompt 列表**：选中某个文件夹（项目），逐条列出该文件夹下所有历史 prompt
2. **全局模糊搜索**：在所有 prompt 中进行关键词模糊搜索，跨文件夹
3. **文件夹内模糊搜索**：进入某个文件夹后，在该文件夹范围内进行模糊搜索

### 设计原则

- **文件夹是一级导航维度**：用户通过选择文件夹来浏览对应的 prompt 历史
- **Prompt 列表是核心内容**：每条 prompt 清晰展示，支持快速浏览
- **搜索是核心交互**：搜索框始终可见，搜索结果即时反馈

### 辅助功能（由开发者设计）

- 统计概览（prompt 总数、项目分布等）
- 对话详情查看
- 时间维度浏览
- UI 布局、主题风格等

## 2. 数据源详解

### 2.1 Prompt 历史 (`~/.claude/history.jsonl`)

每行一个 JSON 对象，记录用户输入的每条 Prompt。

```typescript
interface HistoryItem {
  display: string          // 用户输入的 prompt 文本
  pastedContents: Record<string, string>  // 粘贴的内容
  timestamp: number        // Unix 时间戳 (ms)
  project: string          // 当时的工作目录绝对路径
}
```

### 2.2 项目会话 (`~/.claude/projects/<encoded-path>/*.jsonl`)

路径编码规则：将工作目录路径中的 `/` 替换为 `-`。

```
/Users/you/Desktop/my-project → -Users-you-Desktop-my-project
```

每个 `.jsonl` 文件是一个完整的会话，每行是一条消息：

```typescript
interface ConversationMessage {
  type: 'user' | 'assistant' | 'permission-mode' | 'file-history-snapshot'
  parentUuid: string | null
  uuid: string
  timestamp: string        // ISO 8601 格式
  sessionId: string
  message?: {
    role: 'user' | 'assistant'
    content: string | ContentBlock[]
  }
  cwd?: string
  gitBranch?: string
  version?: string         // Claude Code 版本号
  permissionMode?: string
  isSidechain?: boolean
}

interface ContentBlock {
  type: 'text' | 'tool_use' | 'tool_result'
  text?: string
  name?: string            // tool name
  input?: Record<string, any>
}
```

### 2.3 会话元数据 (`~/.claude/sessions/*.json`)

```typescript
interface SessionMeta {
  pid: number
  sessionId: string
  cwd: string
  startedAt: number        // Unix 时间戳 (ms)
  procStart: string        // 可读时间字符串
  version: string
  kind: 'interactive' | 'headless'
  entrypoint: 'cli' | 'ide'
  status: 'idle' | 'active'
}
```

## 3. Rust 后端 Commands 设计

### 3.1 获取所有 Prompt 历史

```rust
#[tauri::command]
fn get_history(limit: Option<usize>, offset: Option<usize>) -> Result<Vec<HistoryItem>, String>
```

- 读取 `~/.claude/history.jsonl`
- 支持分页（数据量大时避免一次性加载）
- 按时间戳倒序返回

### 3.2 获取项目列表

```rust
#[tauri::command]
fn get_projects() -> Result<Vec<ProjectInfo>, String>
```

- 扫描 `~/.claude/projects/` 下所有目录
- 解码路径名还原为实际工作目录
- 统计每个项目的会话数量和最近活跃时间

```rust
struct ProjectInfo {
    encoded_name: String,    // 编码后的目录名
    display_path: String,    // 还原后的可读路径
    session_count: usize,    // 会话文件数量
    last_active: i64,        // 最近一次对话时间戳
    total_messages: usize,   // 总消息数（估算）
}
```

### 3.3 获取项目下的会话列表

```rust
#[tauri::command]
fn get_project_sessions(project_encoded: String) -> Result<Vec<SessionSummary>, String>
```

- 读取指定项目目录下所有 `.jsonl` 文件
- 提取每个会话的首条用户消息作为标题、起止时间、消息数量

```rust
struct SessionSummary {
    session_id: String,      // 文件名 (UUID)
    first_prompt: String,    // 第一条用户消息（作为标题）
    started_at: String,      // 会话开始时间
    ended_at: String,        // 最后一条消息时间
    message_count: usize,    // 消息总数
    git_branch: Option<String>,
}
```

### 3.4 获取单个对话详情

```rust
#[tauri::command]
fn get_conversation(project_encoded: String, session_id: String) -> Result<Vec<Message>, String>
```

- 读取完整 JSONL 文件
- 过滤掉 `permission-mode`、`file-history-snapshot` 等元数据行
- 只返回 `user` 和 `assistant` 类型的消息

### 3.5 模糊搜索（核心功能）

```rust
#[tauri::command]
fn search_prompts(query: String, project_filter: Option<String>) -> Result<Vec<SearchResult>, String>
```

- **全局模式**（`project_filter = None`）：在 `history.jsonl` 所有 prompt 中模糊匹配
- **文件夹模式**（`project_filter = Some(encoded_path)`）：只搜索指定文件夹下的 prompt
- 模糊匹配策略：子串匹配 + 不区分大小写 + 支持空格分词（多关键词 AND）
- 返回结果包含：匹配的 prompt 文本、所属项目、时间戳、匹配高亮位置

```rust
struct SearchResult {
    display: String,              // prompt 原文
    project: String,              // 所属项目路径
    timestamp: i64,               // 时间戳
    match_ranges: Vec<(usize, usize)>,  // 匹配位置（用于前端高亮）
}
```

### 3.6 获取统计信息

```rust
#[tauri::command]
fn get_stats() -> Result<AppStats, String>
```

```rust
struct AppStats {
    total_prompts: usize,
    total_projects: usize,
    total_sessions: usize,
    prompts_by_day: Vec<(String, usize)>,       // 日期 → 数量
    prompts_by_project: Vec<(String, usize)>,   // 项目 → 数量
    most_active_hours: Vec<(u8, usize)>,        // 小时 → 数量
    first_use_date: String,
    cc_versions_used: Vec<String>,
}
```

## 4. 前端页面与交互设计

### 4.1 整体布局

```

│  [搜索栏：全局模糊搜索]              [全局/文件夹内 切换]  │

│                │                                     │
│  侧边栏         │         主内容区                     │
│  ┌────────────┐│                                     │
│  │ 文件夹列表  ││  根据当前状态显示：                    │
│  │            ││  - 首页：统计概览 + 最近 prompt        │
│  │ /Desktop/  ││  - 文件夹详情：该文件夹的 prompt 列表   │
│  │ /Projects/ ││  - 搜索结果：匹配的 prompt 列表        │
│  │ /Downloads/││                                     │

│  └────────────┘│                                     │
│                │                                     │
└────────────────┴─────────────────────────────────────┘
```

### 4.2 首页（Home）

- 统计卡片：总 prompt 数、项目数、会话数
- 最近的 prompt 列表（全局最新 20 条）
- 最活跃的 Top 5 项目（快捷入口）

### 4.3 文件夹 Prompt 列表（核心页面）

侧边栏点击文件夹后进入：

- 顶部显示当前文件夹路径
- 搜索栏自动切换为"文件夹内搜索"模式
- 下方逐条列出该文件夹的所有 prompt，每条显示：
  - prompt 文本（过长截断，hover 展开）
  - 时间戳（相对时间，如"3天前"）
  - 所属会话标识（可点击展开完整对话）
- 支持按时间排序（新→旧 / 旧→新）

### 4.4 搜索交互

- 搜索框常驻顶部，随时可用
- 输入时实时反馈（debounce 300ms）
- 两种模式通过 toggle 切换：
  - 「全局搜索」：搜所有文件夹的 prompt
  - 「当前文件夹」：只搜选中文件夹内的 prompt（仅在已选文件夹时可用）
- 搜索结果中高亮匹配关键词
- 每条结果显示所属文件夹（全局搜索时）

### 4.5 对话详情（辅助功能）

- 点击某条 prompt 可展开/跳转查看该次对话的完整内容
- 聊天气泡样式展示用户消息和 AI 回复
- 工具调用折叠显示

## 5. 路径编解码逻辑

这是核心工具函数，Rust 侧实现：

```rust
/// 将编码后的目录名还原为可读路径
/// "-Users-you-Desktop-my-project" → "/Users/you/Desktop/my-project"
fn decode_project_path(encoded: &str) -> String {
    // 注意：中文路径的编码方式需要特殊处理
    // 实际编码规则：每个 path segment 前有一个 '-'
    // 需要通过尝试匹配实际存在的路径来还原
}

/// 将实际路径编码为目录名
/// "/Users/you/Desktop/my-project" → "-Users-you-Desktop-my-project"
fn encode_project_path(path: &str) -> String {
    path.replace('/', "-")
}
```

**注意**：中文路径在编码时，中文字符对应的 `-` 数量不等于字符数，需要特殊处理逻辑，建议以扫描 `~/.claude/projects/` 实际目录为准，通过启发式匹配还原路径。

## 7. 性能考量

| 场景 | 策略 |
|---|---|
| history.jsonl 过大（万行级别） | Rust 侧分页读取，前端虚拟列表 |
| 单个对话 JSONL 过大 | 流式读取，先返回摘要再按需加载详情 |
| 项目数过多 | 首次扫描后建立内存索引，后续增量更新 |
| 搜索性能 | Rust 侧做全文匹配，不依赖前端 fuse.js 处理大数据 |

## 8. Tauri v2 权限配置

`src-tauri/capabilities/default.json`：

```json
{
  "identifier": "default",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "fs:allow-read-text-file",
    "fs:allow-exists",
    "fs:allow-read-dir"
  ],
  "scope": {
    "fs": {
      "allow": ["$HOME/.claude/**"]
    }
  }
}
```

只授权读取 `~/.claude/` 目录，不需要写入权限，最小权限原则。

## 9. 注意事项

1. **只读工具**：本项目只读取 Claude Code 数据，绝不修改或删除任何 `~/.claude/` 下的文件
2. **隐私安全**：数据完全本地处理，不联网上传任何对话内容
3. **编码兼容**：中文路径在 macOS (UTF-8) 和 Windows (可能 GBK) 上需要分别处理
4. **版本兼容**：Claude Code 更新可能改变数据格式，需要做好向前兼容的解析逻辑
5. **大文件处理**：部分对话文件可能很大（含 base64 图片等），需要截断或跳过二进制内容
